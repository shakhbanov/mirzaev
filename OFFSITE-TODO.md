# Off-site SEO для doctor-mirzaev.ru

Контентная и техническая работа на сайте сделана. Чтобы добраться до топ-3 в Яндексе и Google по запросам про урологию в Ростове-на-Дону, нужно одновременно прокачать внешние сигналы. Этот файл — список задач **вне сайта**, которые делает врач/маркетолог сам (не код). Срок: 2–4 недели после деплоя сайта.

---

## 1. Поисковые консоли — отправить sitemap

### Яндекс.Вебмастер
1. Зайти на https://webmaster.yandex.ru, добавить сайт `doctor-mirzaev.ru`.
2. Подтвердить владение через DNS-запись (TXT) или мета-тег.
3. Раздел «Файлы Sitemap» → добавить `https://doctor-mirzaev.ru/sitemap.xml`.
4. «Переобход страниц» → отправить вручную: главную и 5 thematic-страниц (prostatitis, prostate-adenoma, kidney-stones, erectile-dysfunction, cystitis).
5. Раздел «Региональность» → указать «Ростов-на-Дону». Это критично для локального ранжирования.
6. Раздел «Турбо-страницы» → опционально подключить (для мобильных).

### Google Search Console
1. https://search.google.com/search-console — добавить ресурс «Доменное свойство» `doctor-mirzaev.ru` (через TXT-запись в DNS).
2. Sitemap → отправить `sitemap.xml`.
3. URL Inspection → проверить и запросить индексацию для главной + 5 thematic.

## 5. Технические TODO в коде сайта

В HTML и JSON-LD оставлены пометки `<!-- TODO -->`. Когда у вас будут реальные данные, просто замените:

- **Лицензия**: в footer всех страниц `Лиц. № <!-- TODO -->` → реальный номер лицензии медицинской деятельности.
- **Цена очного приёма**: в `services.html` (таблица цен) и `faq.html` — заменить «по тарифу клиники» на актуальное значение.
- **ОМС / ДМС**: в `faq.html` — указать конкретные варианты.
- **Даты отзывов**: в schema `reviews.html` — `datePublished` поставлены ориентировочно по тексту отзывов; уточнить с ProDoctorov для точности.

Также:
- **`assets/img/og.jpg` (1200×630)** — отрисовать в Figma/AI или экспортировать из `assets/img/og.svg`. На сайте уже есть SVG-fallback.
- **`assets/img/favicon.ico`** — конвертировать из `favicon.svg` (онлайн-инструмент favicon.io). На сайте сейчас работает SVG favicon (поддерживается всеми современными браузерами).
- **`assets/img/apple-touch-icon.png` (180×180)**, **`icon-192.png`**, **`icon-512.png`** — для PWA. До добавления в `site.webmanifest` ссылки сохраняются, но сам PWA-режим не работает на iOS/Android.
- **`assets/img/doctor.webp`** — конвертировать `doctor.jpg`:
  ```bash
  cwebp -q 82 assets/img/doctor.jpg -o assets/img/doctor.webp
  ```
  В HTML уже подключён `<picture><source srcset=".webp">` с JPG-fallback.
- **Сжать `doctor.jpg`** до ~80–100 КБ:
  ```bash
  jpegoptim --max=82 --strip-all assets/img/doctor.jpg
  # или mozjpeg cjpeg
  ```
